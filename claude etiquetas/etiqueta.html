// html → estructura

Etiquetas de Estructura

Estas son las etiquetas fundamentales que definen la estructura básica de cualquier documento HTML.

Tip: Toda página HTML empieza con <!DOCTYPE html> para indicarle al navegador que use HTML5. Luego viene la estructura base: html → head → body.
<!DOCTYPE> Declaración Documento

No es una etiqueta HTML, sino una instrucción que le dice al navegador qué versión de HTML usar. Siempre va primero.

<!-- Siempre en la primera línea --> <!DOCTYPE html>
<html> Raíz Documento

Es el elemento raíz. Todo el contenido de la página va dentro de esta etiqueta. El atributo lang indica el idioma.

<html lang="es"> ... todo el contenido aquí ... </html>
<head> Cabecera Metadatos

Contiene información sobre la página (metadatos), como el título, estilos, fuentes y configuración. No se muestra en pantalla.

<head> <title>Mi Página</title> <meta charset="UTF-8"> </head>
<meta> Metadatos Cabecera

Define metadatos como la codificación de caracteres, descripción para SEO y configuración para móviles. Es una etiqueta vacía.

<meta charset="UTF-8"> <meta name="viewport" content="width=device-width"> <meta name="description" content="Mi sitio web">
<title> Título Cabecera

Define el título de la página que aparece en la pestaña del navegador y en los resultados de búsqueda (SEO).

<title> Mi Sitio Web — Inicio </title>
<link> Enlace Externo Cabecera

Vincula recursos externos como hojas de estilo CSS o fuentes. El atributo rel define la relación.

<link rel="stylesheet" href="styles.css"> <link rel="icon" href="favicon.ico">
<body> Cuerpo Documento

Contiene todo el contenido visible de la página: textos, imágenes, botones, formularios, etc. Solo puede haber uno por documento.

<body> <h1>¡Hola Mundo!</h1> <p>Mi primer párrafo.</p> </body>
<style> Estilos internos Cabecera

Permite escribir CSS directamente dentro del HTML. Va en el <head>. Para proyectos grandes, es mejor usar archivos CSS externos.

<style> body { background: #f0f0f0; color: #333; } </style>
<div> División Contenedor

El contenedor genérico más usado. Es un elemento de bloque sin significado semántico. Se usa para agrupar y aplicar estilos.

<div class="tarjeta"> <h2>Título</h2> <p>Contenido</p> </div>
<span> Span Contenedor

Contenedor genérico en línea (inline). Se usa para aplicar estilos a una parte del texto sin romper el flujo.

<p>Texto <span style="color:red"> rojo</span> aquí. </p>
// html → texto

Texto y Tipografía

Etiquetas para definir encabezados, párrafos, énfasis y otras variantes de texto.

<h1>-<h6> Encabezados Texto

Definen los títulos de sección. <h1> es el más importante (debe haber solo uno por página). Van de h1 al h6 en orden descendente.

<h1>Título principal</h1> <h2>Subtítulo</h2> <h3>Sección</h3> <h4>Subsección</h4>
<p> Párrafo Texto

Define un párrafo de texto. El navegador añade espacio antes y después automáticamente. Es un elemento de bloque.

<p> Este es un párrafo de texto. Puede tener varias oraciones. </p>
<strong> Negrita semántica Inline

Indica texto de gran importancia. Visualmente aparece en negrita. Tiene significado semántico (a diferencia de <b>).

<p>Esto es <strong>muy importante</strong> para el usuario. </p>
<em> Énfasis Inline

Marca texto con énfasis, visualmente en cursiva. Tiene significado semántico: el texto tiene una entonación especial.

<p>El gato <em>siempre</em> duerme en el sofá. </p>
<br> Salto de línea Inline

Inserta un salto de línea dentro de un párrafo. Es una etiqueta vacía (sin cierre). Úsalo con moderación.

<p> Línea uno.<br> Línea dos. </p>
<hr> Línea horizontal Bloque

Dibuja una línea horizontal divisoria. Representa un cambio temático en el contenido. Etiqueta vacía.

<p>Sección A</p> <hr> <p>Sección B</p>
<blockquote> Cita en bloque Bloque

Representa una cita larga de otra fuente. Normalmente se muestra indentada. El atributo cite puede contener la URL de la fuente.

<blockquote cite="https://fuente.com"> La vida es bella. </blockquote>
<code> Código Inline

Muestra texto como código con fuente monoespaciada. Para bloques de código usa <pre><code>.

<p>Usa <code>console.log()</code> para depurar. </p>
<pre> Texto preformateado Bloque

Muestra el texto con espacios y saltos de línea exactos como están escritos. Ideal para bloques de código.

<pre><code> function suma(a, b) { return a + b; } </code></pre>
<mark> Marcado Inline

Resalta texto como si se usara un marcador fluorescente. Útil para resultados de búsqueda o texto relevante.

<p> Busca la palabra <mark>CSS</mark> en el documento. </p>
<del> / <ins> Borrado / Insertado Inline

<del> muestra texto tachado (eliminado) y <ins> muestra texto subrayado (añadido). Ideal para comparar versiones.

Precio: <del>$100</del> <ins>$75</ins>
<abbr> Abreviatura Inline

Marca una abreviatura o acrónimo. El atributo title muestra la definición completa al pasar el mouse por encima.

<abbr title="HyperText Markup Language" >HTML</abbr>
// html → listas

Listas

Tres tipos de listas para organizar información: ordenadas, desordenadas y de definición.

<ul> Lista desordenada Lista

Crea una lista con viñetas (bullets). El orden de los elementos no importa. Cada elemento va dentro de <li>.

<ul> <li>HTML</li> <li>CSS</li> <li>JavaScript</li> </ul>
<ol> Lista ordenada Lista

Lista numerada donde el orden sí importa. Atributos: start para el número inicial, type para el estilo (1, A, a, I, i).

<ol start="1"> <li>Abrir editor</li> <li>Escribir código</li> <li>Guardar</li> </ol>
<li> Elemento de lista Lista

Define cada elemento dentro de <ul> u <ol>. Puede contener texto, imágenes, o incluso otras listas anidadas.

<ul> <li>Elemento 1</li> <li>Elemento 2 <ul> <li>Sub-elemento</li> </ul> </li> </ul>
<dl> Lista de definición Lista

Lista de términos y sus definiciones. <dt> es el término y <dd> es la descripción. Ideal para glosarios.

<dl> <dt>HTML</dt> <dd>Lenguaje de marcado</dd> <dt>CSS</dt> <dd>Hojas de estilo</dd> </dl>
// html → enlaces e imágenes

Links e Imágenes

Las etiquetas que conectan páginas entre sí y muestran contenido visual.

<a> Enlace (Anchor) Inline

Crea un hipervínculo. href es la URL de destino. target="_blank" abre en nueva pestaña. rel="noopener" por seguridad.

<a href="https://google.com" target="_blank" rel="noopener"> Ir a Google </a>
<img> Imagen Inline

Muestra una imagen. src es la ruta, alt es el texto alternativo (obligatorio para accesibilidad). Es una etiqueta vacía.

<img src="foto.jpg" alt="Descripción" width="300" height="200" >
<figure> Figura Bloque

Agrupa una imagen con su leyenda. <figcaption> es la descripción de la figura. Semánticamente correcto para imágenes con texto.

<figure> <img src="foto.jpg" alt="Un gato"> <figcaption> Mi gato durmiendo </figcaption> </figure>
<picture> Imagen responsiva Bloque

Permite ofrecer diferentes imágenes según el tamaño de pantalla o formato soportado. Muy útil para optimización.

<picture> <source media="(min-width:600px)" srcset="grande.jpg"> <img src="pequena.jpg" alt="Imagen"> </picture>
<map> / <area> Mapa de imagen Inline

Define zonas clicables dentro de una imagen. <map> agrupa las áreas y <area> define cada zona con coordenadas.

<img src="mapa.jpg" usemap="#zonas"> <map name="zonas"> <area shape="rect" coords="0,0,100,100" href="zona1.html"> </map>
// html → tablas

Tablas

Para mostrar datos tabulares estructurados. Nunca uses tablas para maquetación.

<table> Tabla Tabla

Contenedor principal. Dentro van <thead>, <tbody> y <tfoot> para estructura semántica.

<table> <thead>...</thead> <tbody>...</tbody> <tfoot>...</tfoot> </table>
<tr> / <th> / <td> Fila y celdas Tabla

<tr> es una fila, <th> es una celda de encabezado (en negrita), <td> es una celda de datos. colspan y rowspan fusionan celdas.

<tr> <th colspan="2">Encabezado</th> </tr> <tr> <td>Dato 1</td> <td>Dato 2</td> </tr>
<caption> Título de tabla Tabla

Añade un título descriptivo a la tabla. Va justo después de <table>. Mejora la accesibilidad.

<table> <caption> Ventas Enero 2025 </caption> ... </table>
<colgroup> Grupo de columnas Tabla

Permite aplicar estilos a columnas enteras sin repetir código en cada celda. Usa <col> para cada columna.

<colgroup> <col style="background:#f5f5f5"> <col span="2"> </colgroup>
// html → formularios

Formularios

La forma de capturar datos del usuario. Inputs, selects, botones y más.

<form> Formulario Formulario

Contenedor de un formulario. action define a dónde se envían los datos. method puede ser GET o POST.

<form action="/enviar" method="POST"> ...campos... </form>
<input> Campo de entrada Formulario

El elemento más versátil. Cambia con el atributo type: text, email, password, number, checkbox, radio, date, file, color, range...

<input type="text" name="nombre" placeholder="Tu nombre" required> <input type="email" name="correo"> <input type="checkbox" checked>
<label> Etiqueta Formulario

Asocia un texto descriptivo con un campo. Mejora la accesibilidad: al hacer clic en el label, el input recibe el foco.

<label for="email"> Correo electrónico </label> <input type="email" id="email">
<textarea> Área de texto Formulario

Campo de texto multilínea. rows y cols definen el tamaño inicial. El usuario puede redimensionarlo.

<textarea name="mensaje" rows="5" cols="40" placeholder="Escribe aquí..." ></textarea>
<select> Lista desplegable Formulario

Menú de opciones. Cada opción va en <option>. <optgroup> agrupa opciones. multiple permite selección múltiple.

<select name="pais"> <optgroup label="América"> <option value="co">Colombia</option> <option value="mx">México</option> </optgroup> </select>
<button> Botón Formulario

Crea un botón interactivo. type="submit" envía el form, type="reset" lo limpia, type="button" es genérico para JS.

<button type="submit"> Enviar formulario </button> <button type="reset"> Limpiar </button>
<fieldset> Grupo de campos Formulario

Agrupa campos relacionados visualmente con un borde. <legend> le añade un título al grupo. Mejora la accesibilidad.

<fieldset> <legend>Datos personales</legend> <input type="text" name="nombre"> <input type="text" name="apellido"> </fieldset>
<datalist> Lista de sugerencias Formulario

Ofrece sugerencias autocompletadas para un input. El usuario puede escribir o elegir de la lista. Se vincula con list.

<input list="frutas" name="fruta"> <datalist id="frutas"> <option value="Manzana"> <option value="Mango"> </datalist>
// html → multimedia

Multimedia

Incrusta audio, video, iframes y otros contenidos enriquecidos en tu página.

<video> Video Media

Inserta un reproductor de video. Atributos: controls, autoplay, muted, loop, poster. Soporta múltiples fuentes con <source>.

<video controls poster="thumbnail.jpg" width="640"> <source src="video.mp4" type="video/mp4"> Tu navegador no soporta video. </video>
<audio> Audio Media

Inserta un reproductor de audio. Mismos atributos que video. Soporta MP3, OGG, WAV.

<audio controls> <source src="musica.mp3" type="audio/mpeg"> <source src="musica.ogg" type="audio/ogg"> </audio>
<iframe> Marco incrustado Media

Incrusta otra página web o contenido externo (YouTube, Google Maps, etc.). sandbox restringe permisos por seguridad.

<iframe src="https://youtube.com/embed/xyz" width="560" height="315" allowfullscreen ></iframe>
<canvas> Lienzo Media

Área de dibujo programable con JavaScript. Usado para gráficos, juegos, animaciones y visualizaciones de datos.

<canvas id="miCanvas" width="400" height="300" ></canvas>
<track> Pista de subtítulos Media

Añade subtítulos o closed captions a videos y audios. Formato WebVTT. Mejora la accesibilidad enormemente.

<video controls> <source src="video.mp4"> <track src="subtitulos.vtt" kind="subtitles" srclang="es" label="Español"> </video>
<svg> Gráficos vectoriales Media

Dibuja gráficos vectoriales (escalables sin pérdida de calidad) directamente en HTML. Ideal para íconos y logos.

<svg width="100" height="100"> <circle cx="50" cy="50" r="40" fill="blue"/> </svg>
// html → semántico

Etiquetas Semánticas

HTML5 introdujo etiquetas con significado propio. Mejoran el SEO y la accesibilidad.

<header> Encabezado Semántico

Encabezado de la página o de una sección. Normalmente contiene el logo, navegación y título principal del sitio.

<header> <img src="logo.png" alt="Logo"> <nav>...</nav> </header>
<nav> Navegación Semántico

Define un bloque de navegación principal con enlaces. Los lectores de pantalla lo identifican como menú de navegación.

<nav> <ul> <li><a href="/">Inicio</a></li> <li><a href="/blog">Blog</a></li> </ul> </nav>
<main> Contenido principal Semántico

El contenido principal único de la página. Solo debe haber uno. Los lectores de pantalla lo usan para saltar directamente al contenido.

<main> <h1>Título del artículo</h1> <p>Contenido principal...</p> </main>
<article> Artículo Semántico

Contenido independiente y reutilizable: un post de blog, noticia, comentario o tarjeta de producto. Tiene sentido por sí solo.

<article> <h2>Título del post</h2> <p>Contenido...</p> <footer>Autor: Juan</footer> </article>
<section> Sección Semántico

Agrupa contenido relacionado temáticamente. Siempre debe tener un encabezado. Es diferente a <div> porque tiene significado semántico.

<section> <h2>Sobre nosotros</h2> <p>Somos una empresa...</p> </section>
<aside> Contenido lateral Semántico

Contenido relacionado pero secundario, como una barra lateral, publicidad o información complementaria.

<aside> <h3>Artículos relacionados</h3> <ul>...links...</ul> </aside>
<footer> Pie de página Semántico

Pie de página de la página o de una sección. Contiene copyright, links de contacto, créditos y datos del autor.

<footer> <p>© 2025 Mi Empresa</p> <a href="/privacidad"> Privacidad </a> </footer>
<details> Acordeón nativo Semántico

Crea un acordeón colapsable sin JavaScript. <summary> es el título visible. Al hacer clic, muestra u oculta el contenido.

<details> <summary>¿Qué es HTML?</summary> <p> Es el lenguaje de marcado de la web. </p> </details>
<dialog> Diálogo / Modal Semántico

Modal nativo de HTML5. Se abre con JS (showModal()). Maneja automáticamente el foco y la accesibilidad.

<dialog id="modal"> <h2>¡Hola!</h2> <button onclick="modal.close()" >Cerrar</button> </dialog>
<time> Tiempo / Fecha Semántico

Representa una fecha u hora. El atributo datetime da el valor en formato ISO para máquinas y buscadores.

<time datetime="2025-01-15"> 15 de enero de 2025 </time>
<progress> Barra de progreso Semántico

Muestra una barra de progreso nativa. value es el progreso actual y max es el total.

<label>Descargando:</label> <progress value="70" max="100" >70%</progress>
// css → tipografía

Propiedades de Tipografía

Controla fuentes, tamaños, pesos, alineación y espaciado del texto.

font-family Familia tipográfica Texto

Define la fuente del texto. Siempre incluye una fuente genérica de respaldo al final: serif, sans-serif, monospace.

p { font-family: 'Roboto', sans-serif; } code { font-family: 'JetBrains Mono', monospace; }
font-size Tamaño de fuente Texto

Tamaño del texto. Usa rem para accesibilidad (relativo a la raíz), em para relativo al padre, o px para fijo.

h1 { font-size: 2.5rem; } p { font-size: 1rem; } small { font-size: 0.8rem; } /* 1rem = 16px por defecto */
font-weight Grosor de fuente Texto

Controla el grosor del texto: 100 (más delgado) a 900 (más grueso). 400 = normal, 700 = bold.

.titulo { font-weight: 800; } .normal { font-weight: 400; } .delgado { font-weight: 300; }
line-height Altura de línea Texto

El espacio entre líneas. Un valor de 1.5 a 1.7 es ideal para párrafos. Sin unidades es relativo al font-size.

p { line-height: 1.6; } h1 { line-height: 1.1; }
text-align Alineación de texto Texto

Alinea el texto horizontalmente: left, center, right, justify. Para centrar elementos usa flexbox, no text-align.

.banner { text-align: center; } .articulo { text-align: justify; }
letter-spacing Espaciado entre letras Texto

Controla el espacio entre caracteres. Valores positivos amplían, negativos comprimen. Útil para títulos y letras capitales.

.titulo { letter-spacing: -1px; } .etiqueta { letter-spacing: 2px; text-transform: uppercase; }
text-decoration Decoración de texto Texto

Añade o quita líneas decorativas: underline, overline, line-through, none. Muy usado para quitar subrayado de links.

a { text-decoration: none; } .precio-viejo { text-decoration: line-through; }
text-overflow Desbordamiento de texto Texto

Controla cómo se muestra el texto que no cabe. ellipsis añade "..." al final. Requiere overflow:hidden y white-space:nowrap.

.card-title { white-space: nowrap; overflow: hidden; text-overflow: ellipsis; }
// css → modelo de caja

Modelo de Caja (Box Model)

Todo elemento HTML es una caja. Aprende a controlar su tamaño, borde, espaciado interno y externo.

Regla de oro: Siempre añade * { box-sizing: border-box; } a tu CSS. Hace que el padding y border se incluyan dentro del width/height, no se añadan encima.
margin Margen externo Box Model

Espacio fuera del borde del elemento. Separa elementos entre sí. Acepta 1-4 valores (top, right, bottom, left) o auto para centrar.

.caja { margin: 16px; /* todos los lados */ margin: 10px 20px; /* V H */ margin: 0 auto; /* centrar */ margin-top: 24px; /* solo arriba */ }
padding Relleno interno Box Model

Espacio entre el contenido y el borde del elemento. Misma sintaxis que margin. No puede ser negativo ni usar auto.

.boton { padding: 12px 24px; /* V H */ } .tarjeta { padding: 24px; padding-bottom: 32px; }
border Borde Box Model

El borde alrededor del elemento. Definido con grosor, estilo y color. Estilos: solid, dashed, dotted, none.

.caja { border: 2px solid #333; border-top: 4px solid blue; border-radius: 8px; /* redondear */ }
width / height Ancho y alto Box Model

Dimensiones del elemento. min- y max- son las versiones responsive. 100% es relativo al padre, 100vw/vh a la pantalla.

.contenedor { width: 100%; max-width: 1200px; min-height: 100vh; }
display Tipo de visualización Layout

Define cómo se muestra el elemento: block, inline, inline-block, flex, grid, none (ocultar).

.oculto { display: none; } .flex { display: flex; } .grid { display: grid; } .chip { display: inline-block; }
box-shadow Sombra de caja Box Model

Añade sombra al elemento. Valores: X Y blur spread color. Múltiples sombras separadas por coma. inset hace la sombra interior.

.tarjeta { box-shadow: 0 4px 6px rgba(0,0,0,0.1), 0 10px 30px rgba(0,0,0,0.05); }
overflow Desbordamiento Box Model

Controla qué pasa cuando el contenido es mayor que la caja: visible, hidden, scroll, auto. overflow-x/y para cada eje.

.scroll-x { overflow-x: auto; overflow-y: hidden; } .clip { overflow: hidden; }
position Posicionamiento Layout

static (default), relative (relativo a sí mismo), absolute (al padre posicionado), fixed (a la pantalla), sticky.

.padre { position: relative; } .badge { position: absolute; top: -8px; right: -8px; } .nav { position: sticky; top: 0; }
// css → layout

Flexbox y CSS Grid

Las dos herramientas más poderosas del CSS moderno para crear layouts perfectos.

⚡ Flexbox — Distribución en una dimensión
display: flex Activar Flex Flexbox

Activa el modelo de caja flexible en el contenedor. Los hijos inmediatos se convierten en flex items y se alinean en fila por defecto.

.nav { display: flex; align-items: center; justify-content: space-between; }
justify-content Alineación horizontal Flexbox

Controla la distribución de elementos en el eje principal (horizontal por defecto): flex-start, center, flex-end, space-between, space-around.

.centrado { display: flex; justify-content: center; } .separado { display: flex; justify-content: space-between; }
align-items Alineación vertical Flexbox

Alinea los items en el eje cruzado (vertical por defecto): stretch (default), center, flex-start, flex-end, baseline.

.centrado-total { display: flex; justify-content: center; align-items: center; height: 100vh; }
flex-wrap / gap Wrap y espacio Flexbox

flex-wrap: wrap permite que los items se rompan en múltiples líneas. gap define el espacio entre items sin usar margins.

.galeria { display: flex; flex-wrap: wrap; gap: 16px; } .galeria > * { flex: 1 1 200px; }
flex Propiedad flex Flexbox

Shorthand de flex-grow, flex-shrink, flex-basis. flex: 1 hace que el item ocupe todo el espacio disponible.

.sidebar { flex: 0 0 280px; } .main { flex: 1; } /* sidebar fijo, main ocupa el resto */
flex-direction Dirección del flex Flexbox

Cambia el eje principal: row (default, horizontal), column (vertical), row-reverse, column-reverse.

.sidebar { display: flex; flex-direction: column; gap: 8px; }
🗂️ CSS Grid — Distribución en dos dimensiones
display: grid Activar Grid Grid

Activa el layout de cuadrícula bidimensional. Define columnas con grid-template-columns y filas con grid-template-rows.

.grid-3 { display: grid; grid-template-columns: repeat(3, 1fr); gap: 24px; }
grid-template-columns Columnas de la cuadrícula Grid

Define el número y tamaño de columnas. fr es una fracción del espacio libre. repeat() evita repetir. minmax() crea columnas flexibles.

.layout { display: grid; grid-template-columns: 280px 1fr 200px; } .auto-fill { grid-template-columns: repeat(auto-fill, minmax(250px,1fr)); }
grid-column / grid-row Posición del item Grid

Controla qué columnas/filas ocupa un item. span N para que ocupe N celdas. Muy útil para crear layouts asimétricos.

.hero { grid-column: 1 / -1; /* toda la fila */ } .destacado { grid-column: span 2; grid-row: span 2; }
grid-template-areas Áreas nombradas Grid

Define el layout visualmente con nombres. Cada cadena es una fila. Cada palabra es una columna. . es una celda vacía.

.page { display: grid; grid-template-areas: "header header" "aside main" "footer footer"; } header { grid-area: header; } aside { grid-area: aside; }
// css → colores y fondos

Colores y Fondos

Domina los formatos de color, gradientes y propiedades de fondo.

color Color del texto Color

Color del texto. Formatos: nombre (red), hexadecimal (#ff0000), RGB (rgb(255,0,0)), HSL (hsl(0,100%,50%)).

.texto { color: #1f2937; /* hex */ color: rgb(31,41,55); /* RGB */ color: rgba(0,0,0,0.7); /* con opacidad */ color: hsl(220,40%,20%); /* HSL */ }
background Fondo Fondo

Shorthand para todas las propiedades de fondo. Puede ser un color, imagen, gradiente, o una combinación de varios.

.caja { background: #f0f9ff; background: url('imagen.jpg') center/cover no-repeat; }
linear-gradient Gradiente lineal Fondo

Crea una transición de colores en línea recta. Define dirección y colores con sus posiciones opcionales.

.banner { background: linear-gradient( 135deg, #6366f1 0%, #8b5cf6 50%, #ec4899 100% ); }
radial-gradient Gradiente radial Fondo

Gradiente circular que irradia desde un punto central. Muy usado para efectos de luz y profundidad.

.glow { background: radial-gradient( ellipse at center, #6366f1 0%, transparent 70% ); }
--variable Variables CSS Color

Las variables CSS (custom properties) permiten reutilizar valores y crear sistemas de diseño consistentes y fáciles de mantener.

:root { --azul: #3b82f6; --verde: #10b981; --radio: 8px; } .btn { background: var(--azul); border-radius: var(--radio); }
opacity Opacidad Color

De 0 (invisible) a 1 (opaco). Afecta al elemento completo y sus hijos. Para solo el fondo, usa rgba() en background-color.

.overlay { background: #000; opacity: 0.5; } .solo-fondo { background: rgba(0,0,0,0.5); /* el texto no se afecta */ }
filter Filtros visuales Visual

Aplica efectos visuales: blur(), brightness(), contrast(), grayscale(), drop-shadow(), etc.

img:hover { filter: brightness(1.1) saturate(1.2); } .borroso { filter: blur(8px); }
backdrop-filter Filtro de fondo Visual

Aplica filtros al fondo detrás del elemento. El famoso efecto "glassmorphism" se logra con backdrop-filter: blur().

.glass { background: rgba(255,255,255,0.1); backdrop-filter: blur(12px); border: 1px solid rgba(255,255,255,0.2); }
// css → animaciones

Transiciones y Animaciones

Añade movimiento y vida a tu interfaz sin una sola línea de JavaScript.

transition Transición Animación

Anima cambios de propiedades CSS. Define: propiedad, duración, función de tiempo, delay. Se activa en hover, focus, etc.

.boton { background: #6366f1; transition: all 0.3s ease; } .boton:hover { background: #4338ca; transform: translateY(-2px); }
transform Transformación Animación

Transforma el elemento visualmente sin afectar el flujo del documento: translate(), rotate(), scale(), skew().

.icono:hover { transform: rotate(180deg); } .tarjeta:hover { transform: translateY(-8px) scale(1.02); }
@keyframes Fotogramas clave Animación

Define los pasos de una animación compleja. Usa porcentajes (0% a 100%) o palabras clave (from, to).

@keyframes aparecer { from { opacity: 0; transform: translateY(20px); } to { opacity: 1; transform: translateY(0); } }
animation Aplicar animación Animación

Aplica un @keyframes a un elemento. Shorthand: nombre duración timing-function delay iteraciones dirección.

.elemento { animation: aparecer 0.6s ease forwards; } .spinner { animation: girar 1s linear infinite; }
animation-delay Retardo de animación Animación

Retrasa el inicio de la animación. Úsalo con nth-child para crear efectos escalonados (stagger) muy elegantes.

.item:nth-child(1) { animation-delay: 0s; } .item:nth-child(2) { animation-delay: 0.1s; } .item:nth-child(3) { animation-delay: 0.2s; }
:hover / :focus Pseudoclases Selector

Pseudoclases para estados interactivos. :hover al pasar el mouse, :focus al enfocar, :active al hacer click, :visited para links visitados.

a:hover { color: blue; } input:focus { outline: 2px solid #6366f1; } li:nth-child(odd) { background: #f9fafb; }
::before / ::after Pseudoelementos Selector

Insertan contenido virtual antes o después del elemento. Requieren content: "". Muy usados para decoraciones sin HTML extra.

.boton::after { content: " →"; transition: transform 0.2s; } .boton:hover::after { transform: translateX(4px); }
// css → responsivo

Diseño Responsivo

Haz que tu sitio se vea perfecto en cualquier dispositivo con media queries y unidades flexibles.

@media Media Queries Responsivo

Aplica estilos diferentes según el tamaño de pantalla. Enfoque Mobile First: escribe los estilos base para móvil y usa min-width para pantallas mayores.

/* Base: móvil */ .grid { grid-template-columns: 1fr; } /* Tablet: 768px+ */ @media (min-width: 768px) { .grid { grid-template-columns: repeat(2,1fr); } } /* Desktop: 1024px+ */ @media (min-width: 1024px) { .grid { grid-template-columns: repeat(3,1fr); } }
clamp() Valor dinámico Responsivo

Crea valores que escalan fluidamente entre un mínimo y un máximo. clamp(min, ideal, max). Ideal para tipografía fluida sin media queries.

h1 { font-size: clamp(24px, 5vw, 64px); /* mínimo 24px, ideal 5% del viewport, máximo 64px */ }
vw / vh Unidades del viewport Responsivo

vw = 1% del ancho del viewport, vh = 1% del alto. dvh es la versión moderna que excluye la barra del navegador en móvil.

.pantalla-completa { width: 100vw; min-height: 100dvh; } .hero-text { font-size: 5vw; }
@media (prefers-*) Preferencias del usuario Responsivo

Detecta preferencias del sistema: prefers-color-scheme: dark para modo oscuro, prefers-reduced-motion: reduce para menos animaciones.

@media (prefers-color-scheme: dark) { body { background: #0d0d0d; color: #f0f0f0; } } @media (prefers-reduced-motion: reduce) { * { animation: none !important; } }
